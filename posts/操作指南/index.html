博客主题样式操作指南#

本文档记录了 PaperMod 主题的文件结构、配置说明和样式修改方法，方便后续维护和自定义。

一、核心配置文件#

config.yml - 主配置文件#

位置：f:\my-blog\config.yml

这是博客的核心配置文件，控制网站的基本设置和功能开关。

基础配置#

theme: PaperMod              # 主题名称
baseURL: "https://aasd9022.github.io/"  # 网站地址
title: "我的技术博客"         # 网站标题

功能开关（params 部分）#

参数说明默认值
comments评论系统总开关true
defaultTheme默认主题（auto/light/dark）auto
ShowToc显示文章目录true
TocOpen目录默认展开false
ShowReadingTime显示阅读时长true
ShowWordCount显示文章字数true
ShowPostNavLinks上下篇文章导航true
ShowBreadCrumbs面包屑导航true
ShowCodeCopyButtons代码块复制按钮true
ShowLastMod文章最后修改时间true
VisitCount文章访问统计true
ShowShareButtons文章分享按钮true
ShowRelatedPosts相关文章推荐true
disableThemeToggle禁用主题切换按钮false
disableScrollToTop禁用返回顶部按钮false

首页信息展示#

homeInfoParams:
  Title: "欢迎来到我的博客"
  Content: "在这里分享技术心得、编程经验和生活感悟"

社交图标#

socialIcons:
  - name: github
    url: "https://github.com/aasd9022"
  - name: twitter
    url: "https://twitter.com/"
  - name: mail
    url: "mailto:"

支持的社交图标名称：github, twitter, linkedin, mail, rss, youtube, instagram 等。

导航菜单#

menu:
  main:
    - name: 首页
      url: /
      weight: 1    # 权重决定排序顺序
    - name: 归档
      url: /archives/
      weight: 2
    - name: 标签
      url: /tags/
      weight: 3
    - name: 搜索
      url: /search/
      weight: 4

三、关键样式文件详解#

1. theme-vars.css - 主题变量#

位置：themes/PaperMod/assets/css/core/theme-vars.css

定义了全局 CSS 变量，控制整个主题的颜色体系。

浅色模式变量#

:root {
    --gap: 24px;                    # 间距
    --radius: 12px;                 # 圆角大小
    --theme: rgb(255, 255, 255);    # 背景色
    --entry: rgb(255, 255, 255);    # 卡片背景色
    --primary: rgb(30, 30, 30);     # 主要文字颜色
    --secondary: rgb(108, 108, 108); # 次要文字颜色
    --border: rgb(230, 230, 235);   # 边框颜色
    --accent: rgb(66, 153, 225);    # 强调色（链接、按钮等）
    --gradient-start: rgb(66, 153, 225);  # 渐变起始色
    --gradient-end: rgb(156, 39, 176);    # 渐变结束色
}

深色模式变量#

:root[data-theme="dark"] {
    --theme: rgb(20, 20, 25);
    --entry: rgb(30, 30, 38);
    --primary: rgb(230, 230, 235);
    --secondary: rgb(140, 140, 150);
    --border: rgb(45, 45, 55);
    --accent: rgb(96, 179, 255);
    --gradient-start: rgb(96, 179, 255);
    --gradient-end: rgb(192, 108, 242);
}

如何修改主题色#

  1. 修改强调色：找到 --accent 变量，修改为你喜欢的颜色
  2. 修改渐变色：修改 --gradient-start 和 --gradient-end
  3. 修改圆角大小：修改 --radius 值（默认 12px）

2. header.css - 导航栏样式#

位置：themes/PaperMod/assets/css/common/header.css

控制顶部导航栏的外观。

主要样式类#

类名说明
.header导航栏容器，控制毛玻璃效果
.header-nav导航栏内部布局
.logo aLogo 样式，包含渐变效果
.menu a导航链接样式
.menu .active::after当前页面下划线
.theme-toggle主题切换按钮

常见修改#

修改导航栏高度：

/* 在 theme-vars.css 中修改 */
--header-height: 60px;

修改 Logo 字体大小：

.logo a {
    font-size: 24px;  /* 修改此值 */
}

修改导航链接悬停效果：

.menu a:hover::after {
    width: 100%;      /* 下划线展开宽度 */
}

3. post-single.css - 文章详情页#

位置：themes/PaperMod/assets/css/common/post-single.css

控制单篇文章页面的样式。

主要样式类#

类名说明
.post-single文章卡片容器
.post-title文章标题（渐变效果）
.post-content文章正文区域
.post-content h2二级标题样式
.post-content blockquote引用块样式
.post-tags a标签样式
.share-buttons分享按钮区域
.paginav上下篇导航

常见修改#

修改文章标题大小：

.post-title {
    font-size: 40px;  /* 修改此值 */
}

修改引用块样式：

.post-content blockquote {
    border-left: 4px solid var(--accent);  /* 左边框颜色 */
    background: var(--accent-light);       /* 背景色 */
}

修改文章卡片阴影：

.post-single {
    box-shadow: 0 4px 20px var(--shadow);  /* 阴影大小 */
}

4. post-entry.css - 文章列表卡片#

位置：themes/PaperMod/assets/css/common/post-entry.css

控制首页文章列表卡片样式。

主要样式类#

类名说明
.first-entry首篇文章特殊样式
.post-entry普通文章卡片
.home-info首页信息展示区
.entry-header h2卡片标题
.entry-content卡片摘要

常见修改#

修改卡片悬停上浮距离：

.post-entry:hover {
    transform: translateY(-3px);  /* 上浮距离 */
}

修改首篇文章背景渐变：

.first-entry {
    background: linear-gradient(135deg, 
        rgba(66, 153, 225, 0.1), 
        rgba(156, 39, 176, 0.1));
}

5. footer.css - 页脚样式#

位置：themes/PaperMod/assets/css/common/footer.css

控制页脚和返回顶部按钮。

主要样式类#

常见修改#

修改返回顶部按钮颜色：

.top-link {

修改返回顶部按钮位置：

.top-link {

六、自定义样式扩展#

方法一：修改主题 CSS 文件#

直接修改 themes/PaperMod/assets/css/ 下的文件。

优点：直接生效 缺点：主题更新时可能丢失修改

方法二：使用扩展样式文件#

在 assets/css/extended/ 目录创建自定义 CSS 文件。

  1. 创建文件：assets/css/extended/custom.css
  2. 添加自定义样式
  3. Hugo 会自动合并这些样式

优点：不影响主题原文件，便于维护

方法三：覆盖模板文件#

在项目根目录 layouts/ 创建同名模板文件，覆盖主题模板。

例如：创建 layouts/_partials/comments.html 覆盖主题的评论模板。

七、常用操作指南#

修改网站标题和 Logo#

  1. 打开 config.yml
  2. 修改 title 字段
  3. 如需添加 Logo 图片，配置 params.label.icon
params:
  label:
    text: "Home"
    icon: "/logo.png"
    iconHeight: 35

添加新文章#

  1. 在 content/posts/ 创建新的 .md 文件
  2. 添加 Front Matter：
---
title: "文章标题"
date: 2024-06-21T10:00:00+08:00
draft: false
tags: ["标签1", "标签2"]
categories: ["分类"]
---

修改文章列表显示数量#

在 config.yml 添加：

paginate: 10  # 每页显示文章数

添加自定义页面#

  1. 在 content/ 创建文件夹（如 about/）
  2. 创建 _index.md 文件
  3. 在 menu.main 添加导航链接

启用/禁用某篇文章的评论#

在文章 Front Matter 中设置：

comments: false  # 禁用评论

添加文章封面图片#

在文章 Front Matter 中设置：

cover:
  image: "images/cover.jpg"
  alt: "封面描述"
  caption: "图片说明"

十、常见问题#

Q: 修改 CSS 后样式没有变化？#

A: 清理浏览器缓存，或使用 hugo server --disableFastRender 强制完整重建。

Q: 深色模式下颜色不对？#

A: 检查是否同时修改了 :root 和 :root[data-theme="dark"] 两个部分的变量。

Q: 评论系统不显示？#

A: 确保：

  1. GitHub 仓库是公开的
  2. 已安装 Utterances App
  3. 网站已部署到实际域名（本地预览可能不显示）

Q: 如何添加新的社交图标？#

A: 在 config.yml 的 socialIcons 数组中添加新项，PaperMod 支持的图标名称可参考主题文档。

十一、最新修改记录（黑白极简风格改造）#

设计原则#

  • 极简黑白风格：纯黑(#000) + 深灰(#333) + 浅灰(#e5e5e5) 边框
  • 统一字体：PingFang SC, Microsoft YaHei, Helvetica Neue, Arial, sans-serif
  • 去除所有彩色：无渐变、无蓝色、无装饰背景
  • 无圆角：所有圆角设置为 0
  • 极致留白：导航、内容、按钮都使用大量留白

本次修改的清单#

1. config.yml 关键配置#

  • defaultTheme: light - 固定浅色主题
  • disableThemeToggle: true - 移除主题切换按钮
  • disableScrollToTop: true - 移除返回顶部按钮
  • label.text: "我的技术博客" - Logo 改为中文

2. 自定义模板文件#

layouts/list.html - 首页模板

  • 简化结构：Hero 区域 + 文章列表 + 分页
  • 文章摘要使用 truncate 80 限制字数
  • 分页按钮：纯文字 + 箭头（← 上一页 / 下一页 →）

layouts/_partials/breadcrumbs.html - 面包屑

  • 自定义为：只显示"首页"链接
  • 去掉英文 “Home”
  • 隐藏分隔箭头

3. 自定义样式文件#

assets/css/extended/custom.css - 首页自定义样式

  • .hero：标题 22px、描述 14px、按钮 13px

关于预览报错 (net::ERR_ABORTED)#

如果浏览器预览显示 net::ERR_ABORTED 错误：

  • 这是浏览器/插件问题，不是网站问题
  • 实际服务器正常运行
  • 解决方法：在浏览器中手动打开 http://localhost:1313/
  • 或尝试使用无痕模式访问

常用调试命令#

# 启动开发服务器
hugo server --config config.yml --bind 0.0.0.0 --port 1313

# 强制完整重建
hugo server --config config.yml --disableFastRender

# 构建生产版本
hugo --config config.yml

# 清理缓存
hugo --gc
