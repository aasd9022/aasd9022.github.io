    从零开始的Windows专属个人博客搭建全攻略（GitHub Pages+Hugo PaperMod）

    June 20, 2026

    保姆级教程：从零搭建免费Hugo PaperMod个人博客（GitHub永久部署）#

    手把手教小白搭建免费、无广告、高颜值、可自动更新的个人博客，基于 Hugo + PaperMod 主题 + GitHub 全自动部署，无需服务器、无需付费，一次搭建永久使用。

    一、核心原理：先搞懂整套博客运行逻辑#

    整套博客分为本地编写和云端部署两大模块，零基础也能轻松理解：

    1. 本地电脑：安装工具、撰写博客文章、实时预览网页效果

    2. GitHub云端：自动打包网页代码、生成公开专属网址，全网可访问

    三件核心工具（人话翻译）：

    • Git：文件传输工具，负责将本地博客文件同步到GitHub云端仓库（云端U盘）

    • Hugo Extended：静态网页生成工具，将纯文本笔记转换成精美网页

    • GitHub：免费代码托管平台，实现网页自动构建、部署、公网访问

    二、环境准备：电脑必备软件安装（小白零难度）#

    1. 安装 Git（代码传输工具）#

    1. 浏览器搜索「Git官网」，下载 Windows 正式版安装包

    2. 全程默认安装，一路点击 Next，无需修改任何配置

    3. 安装成功标识：桌面/文件夹右键，出现 Git Bash Here、Git GUI Here 选项

    小贴士：命令行无需手动敲代码，全程复制粘贴指令即可，不用死记命令！

    2. 安装 Hugo Extended 扩展版（核心必装）#

    重点提醒：必须安装 Extended 扩展版，普通版缺失主题功能，会导致博客报错！

    1. 打开 Hugo 官方发布页，下载 Windows 端 .zip 压缩包

    2. 解压压缩包，将内部 hugo.exe 单独放入纯英文无中文路径文件夹，示例：F:\hugo

    3. 将该文件夹添加到电脑系统环境变量，添加完成后重启 Git Bash

    4. 验证安装：右键打开 Git Bash，输入 hugo version，输出版本号即安装成功

    3. 注册 GitHub 账号并创建仓库#

    1. GitHub 官网邮箱注册账号，用户名建议简短好记（后续仓库名必须和用户名完全一致）

    2. 新建公开仓库，仓库名称严格遵循固定格式：你的用户名.github.io

    示例：GitHub 用户名为 xiaobai123，仓库名必须为 xiaobai123.github.io，名称错误会直接导致免费域名失效！

    三、本地搭建博客：配置PaperMod高颜值主题#

    步骤1：创建本地博客文件夹#

    在电脑任意磁盘新建纯英文、无空格、无中文文件夹，示例：F:\my-blog

    进入文件夹空白处右键，打开 Git Bash Here，执行指令锁定工作目录：

    cd F:\my-blog

    步骤2：初始化Hugo空白站点#

    复制下方指令粘贴执行，生成博客底层配置文件：

    hugo new site . --force

    步骤3：导入PaperMod主题#

    PaperMod 是 Hugo 主流简约高颜值主题，清爽无冗余特效，适配笔记、技术博客、随笔场景。

    1. 依次执行两行指令，下载并初始化主题子模块：
    git submodule add https://github.com/adityatelange/hugo-PaperMod.git themes/PaperMod
git submodule update --init --recursive

    1. 进入路径 themes/PaperMod/exampleSite，复制文件夹内所有文件

    2. 将复制的文件全部粘贴到博客根目录 my-blog

    3. 根目录找到配置文件 hugo.yml，重命名为 config.yml（解决云端兼容bug）

    步骤4：博客基础个性化配置#

    用记事本打开根目录 config.yml，修改三大核心配置：

    1. 配置核心域名（末尾斜杠 / 不可缺失，否则页面样式失效）：
    baseURL: "https://你的用户名.github.io/"
theme: PaperMod
canonifyURLs: true

    1. 找到 homeInfoParams 区块，自定义博客首页标题、个人简介、头像地址

    2. 按需开启：站内搜索、顶部导航、暗色模式、文章归档等功能

    步骤5：本地实时预览博客#

    Git Bash 执行启动命令，开启本地实时预览：

    hugo server

    浏览器访问 http://localhost:1313 即可查看博客效果，修改内容自动刷新，关闭Git窗口即可停止预览。

    四、云端部署：配置GitHub全自动发布#

    配置完成后，后续只需3行命令，即可自动更新线上博客，无需手动上传文件。

    步骤1：本地绑定GitHub远程仓库#

    依次执行以下指令，关联本地项目与云端仓库：

    # 初始化本地仓库
git init
# 绑定云端仓库（替换为自己的仓库地址）
git remote add origin https://github.com/你的用户名/你的用户名.github.io.git

    步骤2：配置自动部署脚本（GitHub Actions）#

    该脚本可实现：代码推送后，云端自动打包、构建、发布网页。

    1. 根目录新建文件夹：.github/workflows

    2. 在文件夹内新建文件 hugo.yml，粘贴以下完整代码并保存：

    name: Hugo自动构建部署
on: push:
  branches: [ main ]
jobs:
  build-deploy:
    runs-on: ubuntu-latest
    steps:
      - name: 拉取代码（包含主题子模块）
        uses: actions/checkout@v4
        with:
          submodules: true
          fetch-depth: 0
      - name: 安装Hugo扩展版
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: latest
          extended: true
      - name: 构建静态网站
        run: hugo --gc --minify
      - name: 部署到Pages
        uses: peaceiris/actions-gh-pages@v4
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public

    避坑提醒：优先使用这套官方稳定脚本，更换其他脚本极易出现404、部署失败问题。

    步骤3：首次推送全部文件到云端#

    依次执行三行命令，完成首次博客初始化上传：

    # 缓存所有文件修改
git add .
# 提交修改并备注
git commit -m "初始化博客，配置PaperMod主题与自动部署"
# 推送代码到云端
git push --set-upstream origin main

    步骤4：GitHub关键权限配置（解决90%部署报错）#

    推送完成后，进入GitHub仓库后台完成核心设置：

    1. 进入仓库 Settings -Actions - General

    2. 下滑找到 Workflow permissions，勾选 Read and write permissions 并保存（解决403权限报错）

    3. 左侧进入 Pages 部署设置

    4. Build and deployment 选择 Deploy from a branch，分支选 gh-pages，文件夹选择 / (root) 保存

    核心避坑：切勿选择GitHub Actions部署源，会与脚本不匹配，导致网站404！

    步骤5：访问上线博客#

    1. 仓库顶部切换到 Actions，查看自动部署任务

    2. 所有步骤显示绿色对勾即部署成功

    3. 无痕浏览器访问 https://你的用户名.github.io，博客正式上线！

    五、日常使用：发文、改配置、更新博客通用教程#

    1. 新建博客文章#

    执行指令自动生成文章模板，文件默认存放于 content/posts/：

    hugo new posts/我的第一篇博客.md

    打开md文件，顶部核心配置说明：

    title: "我的第一篇博客"
date: 2026-06-20T12:00:00+08:00
draft: true

    重要规则：draft: true 为草稿状态，线上不展示；发布前必须改为 draft: false！

    2. 更新博客通用流程#

    新增文章、修改简介、更换头像等所有更新，统一执行三行命令：

    git add .
git commit -m "更新说明：新增xxx文章/修改首页配置"
git push

    等待1-3分钟，GitHub Actions部署完成，刷新网页即可看到更新。

    3. 更换头像、修改首页信息#

    1. 所有图片素材统一放入根目录static 文件夹

    2. 修改 config.yml 中 homeInfoParams 配置（文字、图片路径）

    3. 执行上述推送命令，自动同步线上内容

    4. 进阶：绑定自定义域名（免费无服务器）#

    1. 阿里云/腾讯云购买域名，完成实名认证（国内必填）

    2. 域名解析后台添加 CNAME 记录，主机记录自定义，记录值为 你的用户名.github.io

    3. 博客根目录新建无后缀文件 CNAME，写入自己的专属域名（例：blog.xxx.com）

    4. 修改 config.yml 的 baseURL 为自定义域名

    5. 推送代码，等待10分钟-24小时解析生效，即可用专属域名访问博客

    六、高频报错踩坑解决方案#

    • 报错：Unable to locate config file 原因：配置文件名为 hugo.yml、重跑旧失败任务 解决：统一改为 config.yml，仅运行最新推送的部署任务

    • 部署403 denied 权限不足 原因：GitHub Actions无读写权限 解决：后台开启 Read and write permissions 权限

    • 部署成功但网页404空白 原因：Pages部署源不匹配、浏览器缓存 解决：切换为gh-pages 分支部署，无痕窗口等待5分钟刷新

    • Git LF/CRLF换行警告 原因：Windows与云端系统换行符冲突（不影响使用） 优化：新建 .gitignore 文件，写入 public/ 忽略本地构建文件夹

    • 新建文章线上不显示 99%原因：未关闭草稿模式，确认 draft: false 再推送

    • 网页样式、图片全部丢失 原因：baseURL 末尾缺失 /，补全斜杠即可

    七、长期使用优化建议#

    1. public 为本地构建产物文件夹，加入 .gitignore 忽略，无需上传云端，精简仓库体积

    2. 禁止直接修改 themes 主题原生文件，自定义样式在根目录新建文件覆盖，后续更新主题不丢失个性化配置

    3. 仓库名必须严格匹配 用户名.github.io，大小写完全一致

    4. 所有图片、附件统一放入 static 文件夹，文章通过相对路径引用

    5. 草稿仅本地可见，正式发布前务必检查草稿状态
